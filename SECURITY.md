# Security Policy

## Reporting Security Vulnerabilities

**DO NOT** create public issues for security vulnerabilities.

Instead, please email: **security@aerolang.dev**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Security Best Practices

### For Users
- Always use the latest version
- Verify downloads from official sources only
- Review generated C++ code before deploying to production

### For Contributors
- Never commit API keys, tokens, or credentials
- Use environment variables for sensitive configuration
- Review dependencies for known vulnerabilities
- Test input validation thoroughly

## Private Repository Access

Some advanced security features are in a private repository:
- Advanced obfuscation techniques
- Commercial-grade optimizations
- Enterprise security modules

Contact us for access if you have a legitimate business need.
